Centering with CSS

Centering Text

Centering text inside of a container is relatively straight forward. We can use text-align: center; to center text horizontally.

HTML

<p>Center this!</p>

CSS

p {
  text-align: center;
}

Center this!

As simple as this may seem, there are still some caveats to consider. Specifically, using text-align: center; will only work if the element being styled has the width to center its content appropriately. If the element is only as wide as the content of the text, our text centering won't appear to do anything.

<p>This isn't centered ðŸ˜¥</p>

CSS

  text-align: center;
  width: fit-content;
}

While the above example appears to be centered quite poorly, as far as the browser is concerned, it is centered. The issue here is not that the text is not centered, rather the element is only as wide as its text, so it looks the same when left, right, or center aligned. In situations where the width of an element is being restricted to the width of its content, other means can be used to center it.

Centering Elements Horizontally

Centering elements has been a finicky challenge over the history of the web. Due to this, a number of different ways to achieve that perfectly centered p inside of its parent div. Here are a number of ways to get something centered horizontally.

Relative Positioning

A valid yet terrible way to center an element is with relative positioning. The reason to avoid this should be clear: relatively positioning an element into the middle of another is incredibly fragile and requires a decent amount of math. This will also destroy any resemblence of responsivness unless even more incredible math is used.

HTML

<div>
  <p>This is sort of centered...</p>
</div>

CSS

p {
  left: calc(50% - 152.1px);
  position: relative;
}

This is sort of centered...

Automatic Margin

Before we had flexbox to do everything for us, a common way to center elements was to use margin: auto;. The way this CSS works is the browser attempts to automatically disperse excess space on the horizontal axis on both sides of the element in the form of margin.

HTML

<div>
  <p>Margin magic!</p>
</div>

CSS

p {
  margin: 0 auto;
}

Margin magic!

Flexbox

The go-to solution for many today when it comes to centering elements is using some good, old flexbox! This kind of problem is what flexbox is built for: displaying elements along a single axis! There are many ways using flexbox to get an element centered. In this example, the element container the element we are centering is centering its content using justify-content: center;.

HTML

<div>
  <p>Flexing into the center!</p>
</div>

CSS

div {
  display: flex;
  justify-content: center;
}

Flexing into the center!

CSS Grid

CSS Grid, a wonderful and buzzy frontend technology, can also be used to center stuff. There are some caveats to using grid though. First, if Internet Explorer has to be (unfortunately) supported, this will be possible, yet miserable. Second, the true power of CSS Grid is realized in two dimensional problems. When centering an element horizontally, a one dimensional problem, it's no better than flexbox.

HTML

<div>
  <p>Center of the "grid"</p>
</div>

CSS

div {
  display: grid;
  justify-items: center;
}

Center of the "grid"

Centering Elements Vertically

Sometimes we're tasked with centering an element vertically as opposed to horizontally. Most of what was covered in the examples for horizontal centering apply to vertical centering as well, except we'll be working in the vertical axis.

Relative Positioning

Just like above, relative positioning can be used to put an element in the center of another element, and it is just as terrible as it is above too!

HTML

<div>
  <p>This still hurts</p>
</div>

CSS

p {
  position: relative;
  top: calc(50% - 25.7px);
}

This still hurts

Margin

While using margin is perfectly valid for centering horizontally, it's not really built to handle centering elements vertically. While it technically can get an element centered vertically, it will also leave that same feeling of terrible that relative positioning tends to give.

HTML

<div>
  <p>This is not ideal</p>
</div>

CSS

p {
  margin: 98px 0;
}

This is not ideal

Flexbox

When using flexbox, centering vertically is trivial. All we have to do is throw align-items: center; into the styling of the container, and we will have a centered element!

HTML

<div>
  <p>Easy with flexbox!</p>
</div>

CSS

div {
  align-items: center;
  display: flex;
}

Easy with flexbox!

CSS Grid

Centering elements verically with CSS Grid is as simple as for flexbox.

HTML

<div>
  <p>Easy with grid too!</p>
</div>

CSS

div {
  align-items: center;
  display: grid;
}

Easy with grid too!

Centering Horizontally and Vertically

Centering an element horizontally and vertically only requires combining some of the methods described above for both. Here's a couple of examples.

HTML

<div>
  <p>Keep scrolling...</p>
</div>

CSS

p {
  left: calc(50% - 55.3px);
  position: relative;
  top: calc(50% - 25.7px);
}

Keep scrolling...

HTML

<div>
  <p>Perfectly centered!</p>
</div>

CSS

div {
  align-items: center;
  display: flex;
  justify-content: center;
}

Perfectly centered!

Conclusion

There are many ways to put text or an element in the middle of another element, and each way has pros and cons worth weighing. For example, as hype as CSS Grid is, if Internet Explorer is (unfortunately) a browser that has to be supported, it's probably not the way to go. Here's a sort of TLDR for what to think about when centering elements

  • Using text-align: center; works for elements that are both displayed as an inline element and have the width available to make meaningful changes to the positioning of the content of the element.
  • Just don't use relative positioning.
  • Using margins to center stuff horizontally is completely valid, and it should be avoided when vertically centering.
  • Flexbox and CSS Grid are both great at doing whatever centering we want!
